Branches¶

Notable branches in Mata.

Main branches¶

master¶

The production release branch of Mata.

devel¶

The developement branch of Mata, providing development releases, used to quickly iterate on new features in Mata without breaking the userland during development. devel is always kept in-sync with master.

Backups¶

Backed-up branches with experiments.

simulation-samo538.bak¶

  • Author: Samuel Lupták (samo538)

  • As part of: Project Practice 1, BIT

  • From: https://github.com/VeriFIT/mata/pull/434

Backup of a NFA reduction algorithm using maximum direct simulation. Pseudo-algorithm is taken from https://github.com/ondrik/iny-fix/blob/master/main.pdf. The experimental results show the algorithm is not outperforming the already existing RT algorithm for simulations.

simulation-reduction-rules-samo538.bak¶

  • Author: Samuel Lupták (samo538)

  • As part of: Project Practice 2, BIT

  • From: https://github.com/VeriFIT/mata/pull/487

Backup of a NFA reduction algorithm enhancement using maximum direct simulation with support for reduction rules.

Detailed Description

Simplified NFA API, used in binding to call NFA algorithms.

In particular, this mostly includes operations and checks, that do not return Automaton, but instead take resulting automaton as pointer (e.g. void f(Nft* result, const Nft& lhs, const Nft& rhs)).

Compute intersection of two NFTs.

Both automata can contain ε-transitions. Epsilons will be handled as alphabet symbols.

Transducers must share alphabets. //TODO: this is not implemented yet. Transducers must have equal values of num_of_levels.

Parameters
[in]lhsFirst NFT to compute intersection for.
[in]rhsSecond NFT to compute intersection for.
[out]prod_mapMapping of pairs of the original states (lhs_state, rhs_state) to new product states (not used internally, allocated only when !=nullptr, expensive).
[in]jump_modeSpecifies if the symbol on a jump transition (a transition with a length greater than 1) is interpreted as a sequence repeating the same symbol or as a single instance of the symbol followed by a sequence of DONT_CARE
[in]lhs_first_aux_stateThe first auxiliary state in lhs. Two auxiliary states does not form a product state.
[in]rhs_first_aux_stateThe first auxiliary state in rhs. Two auxiliary states does not form a product state.
Returns
NFT as a product of NFTs lhs and rhs.

106/**
107 * Create Nft from @p nfa with specified @p num_of_levels.
108 *
109 * This function takes transition from @p nfa as transitions between zero level and the first level of NFT.
110 * All transition between the remaining levels are created based on the function parameters: @p explicit_transitions
111 * and @p next_levels_symbol.
112 *
113 * @param nfa NFA to create NFT from.
114 * @param num_of_levels Number of levels of NFT.
115 * @param explicit_transitions If true, the transitions between levels are explicit (i.e., no jump transitions,
116 * except for the epsilon transitions over all levels).)
117 * @param next_levels_symbol If specified, it is used as a symbol on transitions after the first level.
118 * If not specified, the symbol of the transition is reused.
119 * @return NFT representing @p nfa with @p num_of_levels number of levels.
120 */
121Nft from_nfa_with_levels_zero(const nfa::Nfa& nfa, size_t num_of_levels = DEFAULT_NUM_OF_LEVELS, bool explicit_transitions = true, std::optional<Symbol> next_levels_symbol = {});
122
123/**
124 * @brief Creates Nft from @p nfa with specified @p num_of_levels automatically.
125 *
126 * It assumes that @p nfa is a representation of an nft without jump transitions.
127 * It assign to each state the level based on the distance from the initial state.
128 * For example, if there are 2 levels, the initial states are level 0, the successor
129 * states are level 1, the states after that level 0, etc.
130 *
131 * If you only have one level, then it is more efficient to call the constructor that
132 * takes Nfa as input.
133 *
134 * @throws std::runtime_error if some state should be assigned two different levels
135 * or if the final state is not at level 0.
136 */
137Nft from_nfa_with_levels_advancing(mata::nfa::Nfa nfa, size_t num_of_levels);

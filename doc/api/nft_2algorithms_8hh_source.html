11/**
12 * Concrete NFT implementations of algorithms, such as complement, inclusion, or universality checking.
13 *
14 * This is a separation of the implementation from the interface defined in mata::nft.
15 * Note, that in mata::nft interface, there are particular dispatch functions calling
16 * these function according to parameters provided by a user.
17 * E.g. we can call the following function: `is_universal(aut, alph, {{'algorithm', 'antichains'}})`
18 * to check for universality based on antichain-based algorithm.
19 *
20 * In particular, this includes algorithms for:
21 * 1. Complementation,
22 * 2. Inclusion,
23 * 3. Universality checking,
24 * 4. Intersection/concatenation with epsilon transitions, or,
25 * 5. Computing relation.
26 */

48/**
49 * Inclusion implemented by complementation of bigger automaton, intersecting it with smaller and then it checks
50 * emptiness of intersection.
51 * @param[in] smaller Automaton which language should be included in the bigger one.
52 * @param[in] bigger Automaton which language should include the smaller one.
53 * @param[in] alphabet Alphabet of both automata (it is computed automatically, but it is more efficient to set it if
54 * you have it).
55 * @param[out] cex A potential counterexample word which breaks inclusion
56 * @return True if smaller language is included,
57 * i.e., if the final intersection of smaller complement of bigger is empty.
58 */
59bool is_included_naive(const Nft& smaller, const Nft& bigger, const Alphabet* alphabet = nullptr, Run* cex = nullptr, JumpMode jump_mode = JumpMode::RepeatSymbol);

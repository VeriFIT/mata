13/**
14 * @page nft Nondeterministic Finite Transducers (NFTs)
15 *
16 * @warning Mata provides an experimental (unstable) support for (non-)deterministic finite transducers (NFTs).
17 * The provided interface may change.
18 *
19 * @section nft_design The design of NFTs in Mata
20 *
21 * In Mata, the classical transducers from textbook definitions (where a transition is a tuple of
22 * `(initial state, input symbol, final state, output string)`) are encoded using a simpler data structure for the
23 * transition relation, @c mata::nft::Delta The data structures can directly represent only transitions of the form
24 * `p -a-> s` where `a` is a single symbol or epsilon. A “high-level” transitions of the form `p -abc/def-> q` where
25 * 'abc' is an input word and def is an output word are encoded as a sequence of these 'low level' transitions:
26 * `p -a- p’ -d-> q -b-> q' -e-> r -c-> r' -f-> s` (odd transitions read letters of the input track, even transitions read
27 * letters of the output track, a transition can have epsilon instead of a letter). The primed states are a sort of
28 * internal states, to where the automaton gets after reading a letter on the input tape. States are distinguished by
29 * their “level”. “Normal” states p,q,r,s have level 0, the internal states p’,q’,r’ have in this case the levels 1
30 * (and you could have n-tape transducers where level can be larger than 1).
31 * In Mata, the @c mata::nft::Delta interface is used only for the manual handling of the internal delta representation,
32 * taking only (initial state, input symbol, final state)
33 * We find it useful to think about NFTs in Mata as normal NFAs with a single symbol on a transition (because Mata uses
34 * the same underlying data structure for NFTs as for NFAs) where states are annotated with levels. The 'NFT' states
35 * are the states with levels 0, the other levels are for the internal states. Each NFT transition is a sequence of NFA
36 * transitions, one per tape, between two states with levels 0. Each NFA transition performs a single symbol read on
37 * the tape of the source state.
38 * A sequence of symbols on transitions is not a single word on any tape, but an interleaved description of a single NFT
39 * transition over multiple tapes, e.g., q1(l:0)-{a}->q2(l:1)-{b}->q3(l:2)-{c}->q4(l:0) (where (l:X) describes the level
40 * of the state) is a single 'NFT' transition (from a state with level 0 to another state with level 0) where the NFT
41 * symbol read is a 3-tape symbol ('a', 'b', 'c'), interpreted as 'read 'a' on tape 0, read 'b' on tape 1, and read 'c'
42 * on tape 2.' When you want to read 'af' on a single tape, you need to construct two such transitions where on the
43 * remaining tapes, you read an epsilon symbol.
44 *
45 * @section nft_usage Working with NFTs
46 *
47 * There are some utility functions to simplify creating this NFA-like Delta data structure for NFTs such as
48 * @c mata::nft::Nft::insert_word() (inserting the NFA-like sequence of transitions on different tapes: 'abcd' which means to
49 * read 'ac' on tape 0 and 'bd' on tape 1), and @c mata::nft::Nft::insert_word_by_parts() (inserting word for each tape
50 * separately: {'ac', 'bd'} to achieve the same as in the previous).
51 * If you however want precise control over the created transitions, you can omit using the utility NFT functions and
52 * build your NFT like an NFA using the Nft::Delta::* operations directly, adding a single symbol on a single tape per
53 * add() call. You will have to correctly specify the levels for the states. When working with 2-tape NFTs modelling a
54 * replace operation, we use @c mata::applications::strings::replace namespace with utility functions such as
55 * @c mata::applications::strings::replace::replace_reluctant_regex() (accepting parameters: regex as the input for the tape 0,
56 * and a word as a replacement on tape 1), etc.
57 * The notion of jumps (in the jump mode RepeatSymbol, a mode designed for NFTs) is just an optimization to reduce the
58 * size of the NFT. It is an approach to simplify the data structure for NFTs where you can say that you jump over
59 * several internal states and NFA-like transitions to up to the next state with level 0, each transition in the
60 * sequence reading the same single symbol of the jump. That is useful when, for example, having a NFT reading string
61 * 'abc' on all tapes. You can encode it as a sequence of jumps between states with level 0 as
62 * q0(l:0)-{a}->q1(l:0)-{b}->q2(l:0)-{c}->q3(l:0). Notice that the internal states are not present, but they are implied
63 * by the jump.
64 * Ideally, one should not even have to think about the levels and the intermediate states when properly using the
65 * utility functions (from the @c mata::nft::Nft class and the @c mata::nft namespace in general).
66 *
67 * @see @ref examples/nft.cc example.
68 *
69 * @note If you find some expected NFT operation or utility function missing, do not hesitate to let us know and we will
70 * implement it. The interface for NFTs is not stable yet, so we are open to any and all feedback.
71 */

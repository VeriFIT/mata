21 /**
22 * @brief Creates NFA from regular expression using RE2 parser
23 *
24 * At https://github.com/google/re2/wiki/Syntax, you can find the syntax
25 * of regular expressions with following futher limitations:
26 * 1) If you use UTF8 encoding, the created NFA will have the values of
27 * bytes instead of full symbols. For example, the character Ā whose
28 * Unicode code point is U+0100 and is represented in UTF8 as two
29 * bytes c4 80 will have two transitions, one with c4 followed with
30 * by 80, to encode it.
31 * 2) The created automaton represents the language of the regex and
32 * is not expected to be used in regex matching. Therefore, stuff
33 * like ^, $, \b, etc. are ignored in the regex.
34 *
35 * @sa mata::nfa::builder::create_from_regex()
36 *
37 * @param pattern regex as a string
38 * @param use_epsilon whether to keep epsilon transitions in created NFA
39 * @param epsilon_value symbol representing epsilon
40 * @param use_reduce if set to true the result is trimmed and reduced using simulation reduction
41 * @param encoding encoding of the regex, default is Latin1
42 * @return Nfa corresponding to pattern
43 */

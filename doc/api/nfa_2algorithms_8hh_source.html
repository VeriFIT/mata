28/**
29 * Hopcroft minimization of automata. Based on the algorithm from the paper:
30 * "Efficient Minimization of DFAs With Partial Transition Functions" by Antti Valmari and Petri Lehtinen.
31 * The algorithm works in O(a*n*log(n)) time and O(m+n+a) space, where: n is the number of states, a is the size
32 * of the alphabet, and m is the number of transitions. [https://dl.acm.org/doi/10.1016/j.ipl.2011.12.004]
33 * @param[in] dfa_trimmed Deterministic automaton without useless states. Perform trimming before calling this function.
34 * @return Minimized deterministic automaton.
35 */
36Nfa minimize_hopcroft(const Nfa& dfa_trimmed);

38/**
39 * Complement implemented by determization, adding sink state and making automaton complete. Then it adds final states
40 * which were non final in the original automaton.
41 * @param[in] aut Automaton to be complemented.
42 * @param[in] symbols Symbols needed to make the automaton complete.
43 * @param[in] minimize_during_determinization Whether the determinized automaton is computed by (brzozowski)
44 * minimization.
45 * @return Complemented automaton.

47Nfa complement_classical(const Nfa& aut, const mata::utils::OrdVector<Symbol>& symbols);

49/**

51 * complete. Then it swaps final and non-final states.

58/**
59 * Inclusion implemented by complementation of bigger automaton, intersecting it with smaller and then it checks
60 * emptiness of intersection.
61 * @param[in] smaller Automaton which language should be included in the bigger one.
62 * @param[in] bigger Automaton which language should include the smaller one.
63 * @param[in] alphabet Alphabet of both automata (it is computed automatically, but it is more efficient to set it if
64 * you have it).
65 * @param[out] cex A potential counterexample word which breaks inclusion
66 * @return True if smaller language is included,
67 * i.e., if the final intersection of smaller complement of bigger is empty.
68 */
69bool is_included_naive(const Nfa& smaller, const Nfa& bigger, const Alphabet* alphabet = nullptr, Run* cex = nullptr);
70
71/**
72 * Inclusion implemented by antichain algorithms.
73 * @param[in] smaller Automaton which language should be included in the bigger one
74 * @param[in] bigger Automaton which language should include the smaller one
75 * @param[in] alphabet Alphabet of both automata (not needed for antichain algorithm)
76 * @param[out] cex A potential counterexample word which breaks inclusion
77 * @return True if smaller language is included,
78 * i.e., if the final intersection of smaller complement of bigger is empty.

80bool is_included_antichains(const Nfa& smaller, const Nfa& bigger, const Alphabet* alphabet = nullptr, Run* cex = nullptr);

82/**
83 * Universality check implemented by checking emptiness of complemented automaton
84 * @param[in] aut Automaton which universality is checked
85 * @param[in] alphabet Alphabet of the automaton
86 * @param[out] cex Counterexample word which eventually breaks the universality
87 * @return True if the complemented automaton has non empty language, i.e., the original one is not universal

104/**
105 * @brief Compute product of two NFAs, final condition is to be specified, with a possibility of using multiple epsilons.
106 *
107 * @param[in] lhs First NFA to compute intersection for.
108 * @param[in] rhs Second NFA to compute intersection for.
109 * @param[in] first_epsilons The smallest epsilon.
110 * @param[in] final_condition The predicate that tells whether a pair of states is final (conjunction for intersection).
111 * @param[out] prod_map Can be used to get the mapping of the pairs of the original states to product states.
112 * Mostly useless, it is only filled in and returned if !=nullptr, but the algorithm internally uses another data structures,
113 * because this one is too slow.
114 * @return NFA as a product of NFAs @p lhs and @p rhs with ε-transitions preserved.
115 */
